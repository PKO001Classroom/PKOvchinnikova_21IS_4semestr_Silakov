**Основные функции:**
1.  **Ввод и сохранение достижений:** Пользователь через интуитивный интерфейс вводит название, дату, тип, уровень и описание достижения. Данные сохраняются в локальной базе данных SQLite.
2.  **Просмотр и управление записями:** Во вкладке «Мои достижения» отображается список всех сохранённых записей с возможностью просмотра деталей, удаления и обновления списка.
3.  **Экспорт данных:** Все записи можно экспортировать в структурированный документ Microsoft Word (формат .docx) для формирования портфолио или отчёта.
4.  **Гибкая настройка типов:** Типы достижений (олимпада, сертификат и т.д.) загружаются из внешнего JSON-файла (`types.json`), что позволяет легко их добавлять или изменять без редактирования кода программы.

**Технологический стек:**
*   Язык программирования: **Python**
*   Графический интерфейс: **Tkinter** (библиотека для создания оконных приложений)
*   База данных: **SQLite** (встраиваемая СУБД)
*   Работа с документами: **python-docx** (для создания .docx файлов)
*   Обработка данных: **JSON** (для хранения списка типов достижений)

**Архитектура:** Приложение построено по модульному принципу. Основные операции (работа с БД, экспорт, загрузка типов) вынесены в отдельные функции. Графический интерфейс реализован в виде класса `AchievementsApp`, использующего виджет `Notebook` для организации вкладок.

**Назначение:** Программа предназначена для студентов, школьников, преподавателей и всех, кто хочет вести структурированный учёт своих академических, научных и творческих успехов в одном месте с возможностью быстрого формирования выгрузки.

---

**Глоссарий терминов**

*   **SQLite** — облегчённая встраиваемая система управления базами данных, хранящая всю информацию в одном локальном файле (в данном случае `достижения.db`). Не требует отдельного сервера.
*   **Tkinter** — стандартная библиотека Python для создания графических пользовательских интерфейсов (GUI). Позволяет создавать окна, кнопки, поля ввода и другие элементы.
*   **GUI (Graphical User Interface)** — графический интерфейс пользователя. В данном контексте — это окна и элементы управления программы, с которыми взаимодействует пользователь.
*   **JSON (JavaScript Object Notation)** — простой текстовый формат для хранения и передачи структурированных данных. В программе используется для хранения списка типов достижений в файле `types.json`.
*   **Модуль python-docx** — внешняя библиотека Python, позволяющая создавать, изменять и сохранять документы в формате Microsoft Word (.docx). Используется для функции экспорта.
*   **Combobox (Выпадающий список)** — элемент интерфейса, который позволяет выбрать одно значение из заранее определённого списка. Используется для выбора типа и уровня достижения.
*   **Notebook (Вкладки)** — элемент интерфейса (виджет) в Tkinter, который позволяет организовать несколько панелей (вкладок) в одном окне. В программе это вкладки «Добавить достижение» и «Мои достижения».
*   **База данных (БД)** — организованная структура для хранения, управления и доступа к данным. В программе таблица `достижения` содержит все записи пользователя.
*   **Экспорт данных** — процесс вывода информации из внутреннего формата программы (базы данных) во внешний файл или формат. Здесь — создание документа Word.
*   **CRUD (Create, Read, Update, Delete)** — базовый набор операций для работы с данными: создание, чтение, обновление, удаление. В программе реализованы все эти операции (кроме Update, который можно добавить как улучшение).
