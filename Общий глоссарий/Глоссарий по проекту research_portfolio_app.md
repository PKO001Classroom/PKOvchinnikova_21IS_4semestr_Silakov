## Основные термины:

**Электронное портфолио** — Цифровая система для учёта и систематизации научно-исследовательских достижений студента, включающая публикации, конференции, гранты и другие виды академической активности.

**База данных PostgreSQL** — Реляционная система управления базами данных, используемая для хранения структурированной информации о записях портфолио, соавторах и активности пользователя.

**Markdown** — Упрощённый язык разметки, используемый для форматирования текстовых описаний записей в портфолио с поддержкой заголовков, списков, цитат и кода.

**Graphical User Interface (GUI)** — Графический интерфейс пользователя, созданный с помощью библиотеки Tkinter, обеспечивающий визуальное взаимодействие с системой портфолио.

**Treeview** — Виджет библиотеки Tkinter для отображения данных в табличном формате с возможностью сортировки и выбора записей в интерфейсе приложения.

**Экспорт в Excel/Word** — Функция преобразования данных портфолио в форматы Microsoft Office для создания структурированных отчётов с использованием библиотек openpyxl и python-docx.

**Аналитический отчёт** — Автоматически генерируемый документ, содержащий статистику и визуализацию данных портфолио, включая распределение по типам активности и динамику по годам.

**Модульность** — Архитектурный принцип разделения кода на независимые компоненты (база данных, управление файлами, экспорт данных) для упрощения разработки и поддержки.

**psycopg2** — Библиотека Python для взаимодействия с базой данных PostgreSQL, обеспечивающая выполнение SQL-запросов и управление соединениями.

**Файловый менеджер** — Компонент системы, отвечающий за создание, чтение, обновление и удаление Markdown-файлов, связанных с записями портфолио.

**Соавтор** — Участник научной работы, чьё имя связано с конкретной записью в портфолио через систему связей в базе данных.

**Логирование активности** — Процесс записи действий пользователя (просмотр, создание, обновление записей) для последующего анализа и формирования статистики.

**Валидация данных** — Проверка корректности вводимой информации (название, год, тип записи) перед сохранением в базу данных.

**Кросс-платформенность** — Способность приложения работать в различных операционных системах (Windows, macOS, Linux) благодаря использованию стандартных библиотек Python.

**Генерация графиков** — Создание визуализаций статистических данных с использованием библиотеки Matplotlib для включения в аналитические отчёты.

**Структура БД** — Организация таблиц (entries, coauthors, entry_coauthors, activity_log) и связей между ними для эффективного хранения информации портфолио.
