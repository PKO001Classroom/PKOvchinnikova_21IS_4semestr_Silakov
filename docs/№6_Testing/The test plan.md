## Тест-план (docs/test-plan.md)

### Виды тестов:
1. **Модульное тестирование** - проверка отдельных функций и алгоритмов
2. **Интеграционное тестирование** - проверка взаимодействия модулей
3. **Системное тестирование** - проверка системы в целом
4. **Приемочное тестирование** - проверка соответствия требованиям

### Среда тестирования:
- ОС: Windows 10/11, Linux Ubuntu
- Python 3.8+
- Браузеры: Chrome, Firefox последних версий
- Разрешение экрана: 1920×1080 и выше

### Критерии входа:
1. Завершена разработка всех модулей
2. Выполнено модульное тестирование
3. Документация подготовлена
4. Тестовые данные подготовлены

### Критерии выхода:
1. Все тесты пройдены успешно
2. Критические ошибки отсутствуют
3. Производительность соответствует требованиям
4. Документация актуализирована

### Сценарии тестирования:
1. **Математические расчеты**
   - Проверка корректности вычислений
   - Тестирование на граничных значениях
   - Проверка обработки ошибок

2. **Пользовательский интерфейс**
   - Тестирование удобства использования
   - Проверка отображения на разных разрешениях
   - Тестирование ввода данных

3. **Производительность**
   - Измерение времени расчетов
   - Тестирование с большими объемами данных
   - Проверка использования ресурсов
