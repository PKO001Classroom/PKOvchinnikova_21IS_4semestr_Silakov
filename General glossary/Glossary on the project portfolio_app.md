**Документация: Система электронного портфолио студента-исследователя**

Разработанная система представляет собой десктопное приложение на Python с графическим интерфейсом (GUI) для управления академическими и исследовательскими достижениями студента. Основная цель — централизованное хранение, структурирование и генерация отчётов по различным видам деятельности: публикациям, конференциям, грантам, преподаванию и иным достижениям.

**Архитектура и компоненты:**

1.  **GUI (gui.py):** Основное окно приложения, построенное на библиотеке Tkinter. Включает панели для управления записями, редактирования описаний в формате Markdown, добавления соавторов и генерации отчётов.
2.  **Менеджер базы данных (database.py):** Обеспечивает взаимодействие с PostgreSQL. Отвечает за CRUD-операции (создание, чтение, обновление, удаление) над записями, управление связью «запись-соавтор» и сбор статистики.
3.  **Файловый менеджер (file_handler.py):** Управляет Markdown-файлами, хранящими подробные описания достижений. Автоматически создаёт, читает, обновляет файлы и открывает их во внешнем редакторе.
4.  **Генератор отчётов (exporter.py):** Создаёт сводные отчёты в различных форматах: текстовом (.txt), табличном (.xlsx с использованием openpyxl) и структурированном документе (.docx с использованием python-docx). Является опциональным модулем.
5.  **Хранилище данных:** Состоит из реляционной базы данных (PostgreSQL) для метаданных (название, тип, год) и файловой системы (папка `portfolio_md`) для детализированных описаний.

**Ключевые функции:**
*   Создание и категоризация записей о достижениях.
*   Редактирование описаний с поддержкой Markdown.
*   Привязка неограниченного числа соавторов к каждой записи.
*   Визуализация списка всех записей в табличном виде.
*   Экспорт данных в структурированные отчёты (Excel, Word, текстовый файл).
*   Ведение лога активности (создание, просмотр, обновление записей).

**Требования к окружению:**
*   **Язык:** Python 3.7+
*   **База данных:** PostgreSQL с предустановленной схемой базы `research_portfolio`.
*   **Библиотеки Python:** `psycopg2`, `tkinter` (входит в стандартную библиотеку). Для расширенной функциональности отчётов рекомендуется установить `openpyxl` и `python-docx`.

**Глоссарий (краткий):**

1.  **Запись (Entry)** — основная единица информации в портфолио, представляющая одно достижение (статью, выступление, грант и т.д.).
2.  **Тип записи (Entry Type)** — категория, к которой относится достижение: «Публикация», «Конференция», «Грант», «Преподавание», «Достижение».
3.  **Соавтор (Coauthor)** — соучастник достижения, который может быть привязан к нескольким записям. Хранятся в отдельной таблице БД для устранения дублирования.
4.  **Markdown (.md файл)** — облегчённый язык разметки, используемый для хранения и редактирования подробного описания записи.
5.  **GUI (Graphical User Interface)** — графический интерфейс пользователя, созданный с помощью библиотеки Tkinter, через который происходит всё взаимодействие с системой.
6.  **CRUD (Create, Read, Update, Delete)** — базовые операции для работы с данными, реализованные в модуле `database.py`.
7.  **Генератор отчётов (Report Generator)** — модуль (`exporter.py`), создающий сводные документы на основе данных из БД.
8.  **База данных (Database, БД)** — реляционная база данных PostgreSQL, используемая для хранения структурированных метаданных записей и связей между сущностями.
9.  **Файловый менеджер (File Handler)** — модуль (`file_handler.py`), ответственный за работу с файловой системой: создание, сохранение и открытие текстовых файлов описаний.
10. **Лог активности (Activity Log)** — журнал в БД, фиксирующий действия пользователя (создание, просмотр, обновление записей) для учёта.
