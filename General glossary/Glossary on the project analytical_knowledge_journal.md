# **Документация по проекту "Аналитический журнал знаний"**

## **Общее описание**
Аналитический журнал знаний — это desktop-приложение для систематизации учебных заметок с расширенной аналитикой активности пользователя. Приложение позволяет создавать, редактировать и организовывать конспекты в формате Markdown, категоризировать их, добавлять теги, вести статистику работы и генерировать отчёты в Excel и PDF.

## **Ключевые термины для глоссария:**

1. **Конспект (Note)** – учебная заметка, создаваемая пользователем, хранящаяся в формате Markdown. Содержит заголовок, категорию, теги и путь к файлу.

2. **Категория (Category)** – тематическая группа, к которой относится конспект (например, "Программирование", "Математика").

3. **Тег (Tag)** – ключевое слово для пометки конспекта, упрощающее поиск и фильтрацию (например, "Python", "алгоритмы").

4. **Активность (Activity)** – действие пользователя в системе: создание, редактирование, просмотр конспекта. Логируется в БД.

5. **Markdown** – облегчённый язык разметки, используемый для форматирования текста в конспектах.

6. **PostgreSQL** – система управления реляционной базой данных, используемая для хранения структурированных данных приложения.

7. **Tkinter** – стандартная библиотека Python для создания графического интерфейса (GUI).

8. **Отчёт (Report)** – сформированный документ (Excel или PDF), содержащий статистику пользовательской активности, списки конспектов, теги и графики.

9. **Аналитика (Analytics)** – раздел приложения, отображающий статистику: количество конспектов, тегов, активность по дням, популярные теги.

10. **Экспорт (Export)** – процесс вывода данных в файлы внешних форматов (XLSX, PDF).

11. **Генератор отчётов (Report Generator)** – модуль, ответственный за создание Excel- и PDF-отчётов на основе данных из БД.

12. **Файловый менеджер (File Manager)** – компонент для управления файлами конспектов: создание, чтение, обновление, удаление.

13. **База данных (Database)** – структурированное хранилище информации о конспектах, тегах, категориях и активности.

14. **Graphical User Interface (GUI)** – графический интерфейс пользователя, реализованный с помощью Tkinter.

15. **Вкладка (Tab)** – раздел интерфейса приложения (например, "Конспекты", "Аналитика", "Теги").

16. **Статус-бар (Status Bar)** – нижняя панель интерфейса, отображающая текущее состояние системы или сообщения для пользователя.

17. **Скроллируемый текст (ScrolledText)** – виджет Tkinter для отображения многострочного текста с возможностью прокрутки.

18. **Дерево элементов (Treeview)** – виджет для табличного отображения данных (список конспектов).

19. **Панель управления (Control Panel)** – область интерфейса с кнопками и полями ввода для управления конспектами.

20. **Валидация (Validation)** – проверка корректности введённых данных (например, наличие заголовка конспекта).

## **Итог:**
Проект представляет собой полноценное приложение для ведения учебных заметок с сильным акцентом на аналитику и визуализацию данных. Он объединяет работу с текстом (Markdown), базой данных (PostgreSQL), графическим интерфейсом (Tkinter) и генерацией отчётов (Excel/PDF), что делает его удобным инструментом для студентов, преподавателей и всех, кто систематизирует знания.
